Role: 你是一位精通 Python 和全栈工程化的资深架构师，擅长 Monorepo 架构设计。

Task: 请为我的新项目进行初始化架构设计。

Core Architecture Principles (必须严格遵守):

物理隔离 (Physical Separation):

backend/: 存放所有 Python 相关代码。

frontend/: 存放所有前端 UI 代码。

data/: 独立于代码文件夹。下设 raw/（原始数据）、processed/（处理后数据）、exports/（输出结果）。

后端分层逻辑 (Backend Layers):

packages/ (功能实现层): 所有的子功能必须模块化。例如 packages/crawler、packages/analyser 等。

src/ (调度管理层): 仅存放 main.py 和 API 入口。它通过绝对导入调度各部队，不实现具体业务逻辑。

数据管理与路径规范 (Data & Pathing):

统一路径映射: 在配置模块中利用 pathlib 自动定位根目录下的 data/ 路径。所有读写操作必须基于此基准路径。

读写分离: 业务逻辑不直接操作文件，而是调用统一的存储模块接口。

跨平台兼容: 确保代码在 Windows (本地) 和 Linux (WSL2/Docker) 下路径处理一致。

AI 协作规范:

请先输出一份 完整的目录结构树 (Tree View)。

为项目生成一个 .gitignore 文件，必须包含对 data/ 目录下大数据文件的忽略规则。
